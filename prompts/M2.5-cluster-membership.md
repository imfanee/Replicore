## Goal
Make the cluster **dynamic and operable**: add/remove nodes with zero downtime
from any node, propagate membership safely across the mesh, bootstrap a joining
node while it keeps taking live writes, and provide a CLI that inspects and
controls the local node and queries the whole cluster remotely.

## In scope (RSD addendum)

  gossip dissemination, trust-gated admission, propagation, lifecycle states,

- FR-1310/1311: join-time bidirectional reconcile + parallel live sync via a

## Mandated design (these four decisions are fixed — do not improvise them)

### 1. Two configs: human-owned intent vs agent-owned roster
- `replicore.toml` (intent): local identity, shares, limits, **trust anchors**,
  **seed peers**. Hand-edited only. The daemon **must never write it**.
- A separate agent-managed **roster** file: the dynamically-learned member set.
  The daemon owns it exclusively.
- `reload` reads the intent file, merges with the roster, and applies. This
  avoids the edit-race that would occur if the daemon rewrote the file an
  operator also edits.

### 2. Roster = versioned OR-Set + gossip
- Each roster entry carries `{node_id, endpoint, cert_fingerprint, epoch}`.
  Removals leave **tombstones** (so stale gossip cannot resurrect a removed
  node). Concurrent add/remove from different nodes must **converge
  deterministically** (observed-remove set semantics), not fight.
- Disseminate liveness via **SWIM-style gossip**; disseminate identity via the
  versioned roster. A handful-of-nodes cluster does **not** need Raft.

### 3. Trust gate — announcement is not authorization
- A learned peer is admitted to the data path **only after** its certificate
  validates against a configured trust anchor (or an explicitly approved
  fingerprint). Membership-changing operations are signed by an **admin key**.
- Flow (FR-1306): an existing node announces a **candidate** member; every peer
  **independently verifies trust** before connecting. "Connected successfully
  without error" must mean "mTLS validated against my trust anchor," never
  "another node vouched for it." A single compromised node must not be able to
  inject peers into the mesh.

### 4. Join frontier — bootstrap while live
- Lifecycle: `JOINING → SYNCING → ACTIVE`.
- On join, compare local inventory vs mesh via Merkle reconcile (FR-1310) to
  compute the bidirectional initial sync set.
- Bulk catch-up brings the node to a **version-vector frontier**. Live ops with
  VVs **beyond** the frontier apply on top; ops already in the bootstrap are
  deduped by op-id / VV. This is base-backup + WAL-streaming; M1 op-log
  idempotency + version vectors make it safe, but the **handoff at the frontier
  must be coded deliberately** or writes are lost/double-applied across it.
- The node serves its **own namespace immediately**, but is authoritative for
  shared reconcile only at `ACTIVE`.

### Control plane shape
- `replicorectl` talks to the local agent over an authenticated Unix domain
  socket. Read commands are local; `status --all` / queries fan out over the
  **mesh control channel**, aggregate, and return partial results (marking
  unreachable nodes) within a bounded timeout — never hang.
- `config reload` is atomic (reject invalid candidate, leave running config
  untouched). `config diff` classifies hot-appliable vs restart-required and
  reports errors. `--json` on read commands.

## Deliverables
- `membership` module (roster OR-Set + epochs + tombstones), `gossip` (SWIM),
  `join` (lifecycle + frontier handoff), trust-gated admission in `peer`,
  `config` (intent/roster split, validate, diff, atomic reload),
  `control` (UDS server + fan-out), `replicorectl` binary.
- Property test: concurrent add/remove from multiple nodes converges to one
  roster. Integration test on a 3→4 node join (and a removal) on the rig.

## Exit criteria (must be demonstrated — addendum AC 8–12)
1. Add a node to one node's intent config + reload → it joins, membership
   propagates, all peers converge their roster and connect, with **zero**
   interruption to ongoing replication or local I/O. No restarts.
2. Joining node bootstraps via reconcile **while taking live writes**; final
   state converges with no lost/duplicated ops across the frontier.
3. `config diff` reports added/removed/changed peers correctly; an invalid
   candidate is rejected on reload with the running config intact.
4. A peer whose cert does not chain to the trust anchor is refused admission
   even when announced by a trusted node.
5. `replicorectl status --all` from any node aggregates all nodes and degrades
   to marked partial results under a partition.
6. `cargo clippy --all-targets -- -D warnings` clean.

## Reviewer checklist (human gate — the membership traps live here)
- [ ] **Split-brain membership**: drive concurrent `member add` of *different*
      nodes at *two* nodes simultaneously; confirm rosters converge identically
      (OR-Set epochs/tombstones), not diverge.
- [ ] **Tombstone correctness**: a removed node announced by stale gossip is NOT
      re-added.
- [ ] **Trust gate**: an announced peer with an untrusted cert never opens a data
      connection; verify by injecting a bogus candidate.
- [ ] **Admin-key auth**: an unsigned/forged membership change is rejected.
- [ ] **Frontier handoff**: write continuously to a peer during a join; after
      `ACTIVE`, diff both nodes — zero missing, zero duplicated. Test a `kill -9`
      of the joiner mid-bootstrap and confirm clean resume.
- [ ] **Atomic reload**: feed a syntactically valid but semantically broken
      config (e.g. peer with no reachable endpoint) and confirm running config
      is untouched and the error is reported.
- [ ] **Zero downtime**: a transfer in progress during add/remove/reload
      completes uninterrupted; local reads/writes never block.
- [ ] **Fan-out never hangs**: partition one node; `status --all` returns within
      the timeout with that node marked unreachable.
- [ ] Daemon never writes the intent file (inspect for any write path to it).
